📦 CSS LAYOUT & DISPLAY (DETAILED)

Block, Inline, Overflow & Visibility with Types + Real Life Examples
1. BLOCK ELEMENTS
Definition: Block elements take full width and always start on a new line.
Behavior: They stack vertically (one below another).
📌 Real Life: One person sitting alone on full bench
div { display: block; }
2. INLINE ELEMENTS
Definition: Inline elements take only required space and stay in same line.
Behavior: Width & height cannot be applied properly.
📌 Real Life: Students sitting side by side on same bench
span { display: inline; }
3. INLINE-BLOCK ELEMENTS
Definition: Mix of block + inline behavior.
Behavior: Stay in line + allow width/height.
📌 Real Life: Small boxes arranged in a row
4. DIV VS SPAN
Div: Block element used for layout structure.
Span: Inline element used for styling text.
📌 Real Life: Div = Room, Span = Decoration inside room
5. OVERFLOW PROPERTY (WITH TYPES)
Definition: Controls what happens when content is too big for its container.
Real Life Example: Water overflowing from a glass
Types:
/* 1. hidden - hides extra content */ div { overflow: hidden; } /* 2. scroll - adds scroll bars */ div { overflow: scroll; } /* 3. auto - adds scroll only if needed */ div { overflow: auto; } /* 4. visible - default (content shows outside) */ div { overflow: visible; }
6. VISIBILITY PROPERTY (WITH TYPES)
Definition: Controls whether element is visible or not, but keeps space.
Real Life Example: Object is covered with cloth but still exists
Types:
/* visible - default */ div { visibility: visible; } /* hidden - hides but space remains */ div { visibility: hidden; } /* collapse - used for table rows */ div { visibility: collapse; }
